PHASE 5: ADMIN DASHBOARD – CORPORATE RIDES
=================================================

1. Create a dedicated Admin section: “Corporate & Scheduled Rides”.

Admin capabilities:
- View all corporate organizations.
- See organization profile:
  - Organization name
  - Account status (Active / Suspended)
  - Wallet balance
  - Total rides
  - Monthly usage
- View list of members (riders linked to organization).
- View trip history (filterable by date, member, status).

2. Corporate wallet controls:
- Admin can:
  - View wallet balance
  - View wallet transaction history
  - Freeze wallet if suspicious activity detected
- Admin CANNOT:
  - Edit balances manually
  - See payment gateway secrets

3. Ride management:
- View upcoming scheduled corporate rides.
- View completed corporate rides.
- See cancellation reasons and timestamps.
- See cancellation fees applied (without revealing internal splits).

4. Permissions:
- Corporate admins can book rides.
- Corporate members can only ride.
- Admin can revoke or restore access.

PHASE 6: CANCELLATION FEE AMOUNTS (STRATEGIC)
=================================================

5. Set default cancellation fee values (configurable in Admin):

Recommended Nigeria defaults:
- Grace period after driver accepts: 3 minutes.
- Cancellation fee after grace period (driver en route): ₦500.
- Cancellation after driver arrives: ₦1,000.

6. Rules:
- Fee amounts stored in Admin Settings.
- Fees may be adjusted per city/zone in future.
- Never display fee breakdown or calculations to users.
- UI text must say:
  “A cancellation fee may apply.”

7. Driver compensation:
- Fee is credited to driver wallet or trip earnings.
- Do NOT label as “commission” anywhere.

PHASE 7: BANK TRANSFER WALLET FUNDING
=================================================

8. Add Bank Transfer as a wallet funding option:

Supported flow:
- User selects “Add Funds” → “Bank Transfer”.
- App generates:
  - Unique virtual account number OR
  - Bank instructions with reference code.
- Transfer is matched automatically to wallet.

9. Bank transfer rules:
- Funds credited after confirmation.
- Pending transfers shown as “Processing”.
- Failed or unmatched transfers flagged for support review.
- No direct bank-to-driver payments.

10. Supported users:
- Riders
- Drivers
- Corporate accounts

11. Admin tools:
- View pending bank transfers.
- Approve or flag failed transfers.
- Audit trail required.
